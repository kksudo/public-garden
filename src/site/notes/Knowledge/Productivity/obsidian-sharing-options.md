# Как шарить Obsidian заметки с коллегами

> Задача: делиться отдельными заметками с коллегами, как в Notion.

---

## Варианты — быстрое сравнение

| Метод | Бесплатно | Сложность | Частный доступ | Редактирование | Лучше для |
|---|---|---|---|---|---|
| **GitHub Gist** | ✅ | Минимальная | ❌ (публично) | ❌ | Разовая отправка кода/текста |
| **GitHub Pages + Digital Garden** | ✅ | Средняя | ❌ (публично) | ❌ | Публичный сайт-заметки |
| **Obsidian Publish** | ❌ $10/мес | Минимальная | ✅ (пароль) | ❌ | Полноценный публичный/закрытый сайт |
| **Quartz (self-hosted)** | ✅ | Высокая | ✅ (настройки сервера) | ❌ | IT-команда, свой сервер |
| **Shared GitHub repo** | ✅ | Средняя | ✅ (приватный репо) | ✅ (через PR) | Команда разработчиков |
| **Notion import** | ✅ | Минимальная | ✅ | ✅ | Если коллеги уже в Notion |
| **Markdown → Confluence** | ✅/платно | Средняя | ✅ | ✅ | Enterprise команды |

## Детальный разбор

### 1. GitHub Gist — для разовых случаев
**Когда использовать:** "Вот статья, посмотри" — отправил ссылку и забыл.

- Создаёшь gist на https://gist.github.com
- Копируешь markdown файл → рендерится красиво
- Делаешь **Secret Gist** → ссылка работает, но не индексируется
- **Минус:** нет структуры, нет wikilinks, нельзя шарить папку

**Подходит для:** 1-2 заметки, не требующих обновления.

---

### 2. Digital Garden (плагин) + GitHub Pages — бесплатный сайт
**Когда использовать:** хочешь публичный сайт из избранных заметок.

- Плагин: `oleeskild/obsidian-digital-garden`
- Помечаешь нужные заметки: `dg-publish: true` во frontmatter
- Остальные заметки остаются **приватными** в vault
- Деплой в Vercel/Netlify — бесплатно

**Плюсы:**
- Полностью бесплатно
- Поддерживает Dataview, Excalidraw, backlinks, граф
- Выглядит как полноценный сайт

**Минусы:**
- Настройка занимает 30-60 минут
- Публично (без пароля из коробки)
- Не совместное редактирование

**Ссылки:** https://dg-docs.ole.dev

---

### 3. Obsidian Publish — официальный, платный
**Когда использовать:** хочешь без возни, с паролем для закрытого доступа.

- $10/мес
- Пароль на весь сайт или на отдельные страницы
- Поддержка всех Obsidian фич из коробки

**Минус:** только для чтения, коллеги не могут редактировать.

---

### 4. Приватный GitHub репо — для команды разработчиков ⭐
**Когда использовать:** коллеги умеют работать с git.

- Создаёшь приватный репо `team-notes`
- Добавляешь коллег как Collaborators
- Каждый клонирует себе и открывает как Obsidian vault
- Изменения через git pull/push

**Плюсы:**
- Бесплатно
- Полный доступ к редактированию
- История изменений
- Уже у тебя настроен Obsidian Git

**Минусы:**
- Коллеги должны уметь git
- Конфликты при одновременном редактировании одного файла

**Вариант для технической команды PRGate** — оптимально.

---

### 5. Quartz — self-hosted, продвинутый
**Когда использовать:** хочешь полный контроль, своя аутентификация.

- https://quartz.jzhao.xyz
- Статический сайт из vault, деплоится на свой сервер
- Можно добавить HTTP Basic Auth или закрыть за VPN

**Минус:** требует настройки сервера.

---

### 6. Экспорт в Notion (парадокс, но работает)
Если коллеги уже сидят в Notion:
- Notion умеет импортировать Markdown
- Экспортируешь нужные заметки → загружаешь в Notion
- Коллеги редактируют в Notion, ты синхронизируешь обратно

**Плагины для двусторонней синхронизации:** `obsidian-to-notion`, `NotionNext`

## Рекомендации для тебя

### Сценарий A: "Поделиться одной заметкой быстро"
→ **GitHub Gist** (Secret) — 2 минуты, ссылка готова

### Сценарий B: "Публичные заметки по теме (как блог)"
→ **Digital Garden** + Vercel — бесплатно, выглядит профессионально

### Сценарий C: "Команда PRGate читает и редактирует"
→ **Приватный GitHub репо** — добавить коллег, клонировать vault

### Сценарий D: "Без возни, готов платить $10/мес"
→ **Obsidian Publish** — настройка за 10 минут

## Быстрый старт: Digital Garden (рекомендую попробовать)

1. Установи плагин `Digital Garden` в Obsidian (`Cmd+,` → Community Plugins)
2. Создай репо на GitHub: `my-digital-garden`
3. В плагине укажи GitHub token + имя репо
4. Добавь в любую заметку во frontmatter: `dg-publish: true`
5. В плагине нажми "Publish" → автоматически деплоится на Vercel
6. Получаешь ссылку — шаришь коллегам

**Время настройки:** ~45 минут
