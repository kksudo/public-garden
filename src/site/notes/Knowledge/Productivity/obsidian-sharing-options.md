# Как шарить Obsidian заметки с коллегами

> Задача: делиться отдельными заметками с коллегами, как в Notion.

---

## Мой кейс: бесплатный стек ⭐

**Использую:** Obsidian (бесплатно) + Digital Garden плагин (бесплатно) + Vercel (бесплатно)

- Публичный сайт: https://notes.kazakov.xyz
- Кастомный домен через Cloudflare
- Автоматический деплой при каждом `Publish` из Obsidian
- Полностью бесплатно, выглядит профессионально

**Как работает:**
1. В Obsidian помечаю заметки `dg-publish: true` во frontmatter
2. Нажимаю "Publish" в плагине Digital Garden
3. Плагин пушит изменения в GitHub репо
4. Vercel автоматически деплоит сайт
5. Коллеги получают ссылку на публичную заметку

**Время настройки:** ~45 минут (один раз), дальше — один клик.

## Детальный разбор

### 1. Digital Garden + Vercel — бесплатный публичный сайт ⭐ (мой кейс)

**Когда использовать:** хочешь публичный сайт из избранных заметок, полностью бесплатно.

**Стек:**
- Obsidian (бесплатно) — локальный редактор
- Плагин `oleeskild/obsidian-digital-garden` (бесплатно)
- GitHub репо (бесплатно) — хранилище
- Vercel (бесплатно) — хостинг и автоматический деплой

**Как работает:**
1. Устанавливаешь плагин Digital Garden в Obsidian
2. Создаёшь GitHub репо (например, `my-digital-garden`)
3. Настраиваешь плагин: GitHub token + имя репо
4. Помечаешь нужные заметки: `dg-publish: true` во frontmatter
5. Остальные заметки остаются **приватными** в vault
6. Нажимаешь "Publish" в плагине → автоматически деплоится на Vercel
7. Получаешь ссылку (можно подключить кастомный домен)

**Плюсы:**
- Полностью бесплатно (Obsidian + плагин + GitHub + Vercel)
- Поддерживает Dataview, Excalidraw, backlinks, граф знаний
- Выглядит как полноценный сайт
- Автоматический деплой при каждом обновлении
- Можно подключить кастомный домен (через Cloudflare или напрямую)
- SEO-friendly структура

**Минусы:**
- Настройка занимает 30-60 минут (один раз)
- Публично (без пароля из коробки, но можно закрыть через Vercel настройки)
- Не совместное редактирование (только чтение для коллег)

**Документация:** https://dg-docs.ole.dev

**Мой пример:** https://notes.kazakov.xyz

### 2. GitHub Gist — для разовых случаев

**Когда использовать:** "Вот статья, посмотри" — отправил ссылку и забыл.

- Создаёшь gist на https://gist.github.com
- Копируешь markdown файл → рендерится красиво
- Делаешь **Secret Gist** → ссылка работает, но не индексируется поисковиками
- **Минус:** нет структуры, нет wikilinks, нельзя шарить папку

**Подходит для:** 1-2 заметки, не требующих обновления.

---

### 3. Obsidian Publish — официальный, платный

**Когда использовать:** хочешь без возни, с паролем для закрытого доступа.

- $10/мес за vault
- Пароль на весь сайт или на отдельные страницы
- Поддержка всех Obsidian фич из коробки
- Настройка за 10 минут

**Минус:** только для чтения, коллеги не могут редактировать.

---

### 4. Приватный GitHub репо — для команды разработчиков

**Когда использовать:** коллеги умеют работать с git.

- Создаёшь приватный репо `team-notes`
- Добавляешь коллег как Collaborators
- Каждый клонирует себе и открывает как Obsidian vault
- Изменения через git pull/push

**Плюсы:**
- Бесплатно
- Полный доступ к редактированию
- История изменений
- Уже у тебя настроен Obsidian Git

**Минусы:**
- Коллеги должны уметь git
- Конфликты при одновременном редактировании одного файла

**Вариант для технической команды** — оптимально.

---

### 5. Quartz — self-hosted, продвинутый

**Когда использовать:** хочешь полный контроль, своя аутентификация.

- https://quartz.jzhao.xyz
- Статический сайт из vault, деплоится на свой сервер
- Можно добавить HTTP Basic Auth или закрыть за VPN

**Минус:** требует настройки сервера и технических знаний.

### 6. Экспорт в Notion (парадокс, но работает)

Если коллеги уже сидят в Notion:
- Notion умеет импортировать Markdown
- Экспортируешь нужные заметки → загружаешь в Notion
- Коллеги редактируют в Notion, ты синхронизируешь обратно

**Плагины для двусторонней синхронизации:** `obsidian-to-notion`, `NotionNext`

## Быстрый старт: Digital Garden + Vercel (рекомендую)

**Полная настройка за 45 минут:**

1. **Установи плагин** `Digital Garden` в Obsidian
   - `Cmd+,` (или `Ctrl+,`) → Community Plugins → Browse
   - Ищи `Digital Garden` от `oleeskild`
   - Установи и включи

2. **Создай GitHub репо**
   - Создай новый репо на GitHub (например, `my-digital-garden`)
   - Публичный или приватный — на выбор

3. **Настрой плагин**
   - Открой настройки Digital Garden в Obsidian
   - Укажи GitHub token (создай Personal Access Token с правами `repo`)
   - Укажи имя репо (owner/repo-name)
   - Выбери платформу деплоя: **Vercel** (рекомендую)

4. **Подключи Vercel**
   - При первом "Publish" плагин предложит подключить Vercel
   - Авторизуйся через GitHub
   - Vercel автоматически создаст проект и настроит деплой

5. **Опубликуй первую заметку**
   - Добавь в любую заметку во frontmatter: `dg-publish: true`
   - В плагине нажми "Publish"
   - Подожди 1-2 минуты — сайт автоматически задеплоится

6. **Получи ссылку**
   - Vercel даст ссылку вида `my-digital-garden.vercel.app`
   - Можно подключить кастомный домен в настройках Vercel

**Дальше:** просто добавляй `dg-publish: true` в нужные заметки и нажимай "Publish" — всё обновляется автоматически.

**Время настройки:** ~45 минут (один раз), дальше — один клик на публикацию.
