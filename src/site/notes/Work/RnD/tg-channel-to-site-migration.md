# Telegram Channel → Сайт/Магазин — Сервис-идея

> **Контекст:** владельцы TG-каналов с аудиторией и продажами хотят переехать на полноценный сайт или интернет-магазин. Не знают как, не хотят переплачивать разработчикам, боятся потерять аудиторию.

---

## Проблема

TG-канал как витрина — это временное решение:
- Нет SEO, нет органического трафика
- Нельзя нормально принимать оплату (только ручные переводы / боты)
- Алгоритмы Telegram могут изменить охваты в любой момент
- Нет аналитики, нет CRM, нет воронки
- Невозможно масштабировать без личного участия

**Типичный клиент:** малый бизнес с 1k–50k подписчиков в TG, продающий товары/услуги через личку или боты.

---

## Чем это отличается от "просто сделать сайт"

**Обычный сайт с нуля:**
- Контент создаётся заново — часы/дни работы
- Аудитория не знает о новом сайте — нужна отдельная кампания
- Владелец привязан к одному каналу (сайт OR telegram)
- Потеря накопленного доверия подписчиков

**TG → Site миграция:**
- Контент уже есть — переносим, структурируем, SEO-оптимизируем
- TG-канал остаётся активным и ведёт на сайт → нулевая потеря аудитории
- Два канала работают вместе: TG для прогрева → сайт для конверсии
- Подписчики видят знакомый бренд в новом месте → выше доверие

**Ключевой инсайт:**
> TG-канал — это уже готовый контент-архив + тёплая аудитория. Задача не "сделать сайт", а "монетизировать то что уже есть".

---

## Сроки переноса

| Объём | Описание | Срок |
|---|---|---|
| Мини (до 50 постов) | Лендинг + каталог до 20 товаров | 3–5 дней |
| Стандарт (50–200 постов) | Магазин до 100 товаров + блог | 7–10 дней |
| Большой (200+ постов) | Полный магазин + контент-база + автоматизация | 14–21 день |

**Что занимает больше всего времени:**
- Структурирование хаотичных постов в каталог (30% времени)
- Настройка оплаты и тесты (20% времени)
- Контент: описания, фото, SEO-тексты (40% времени)
- Деплой и настройка домена (10% времени)

---

## УТП сервиса

**"Переезд за 2 недели без потери аудитории"**

- Переносим каталог/контент → сайт
- Настраиваем оплату (ЮKassa, Tinkoff, Stripe)
- Telegram-бот остаётся как канал уведомлений → аудитория не теряется
- SEO-базис с первого дня
- Обучаем управлять самостоятельно

---

## Форматы продукта

### Tier 1 — Лендинг + приём заявок ($500–800)
- Одностраничник на Tilda/Framer
- Форма заявки + уведомления в TG
- Миграция описаний и фото из канала
- Срок: 5–7 дней

### Tier 2 — Интернет-магазин ($1500–3000)
- Shopify / WooCommerce / самописный на Next.js
- Каталог, корзина, оплата
- Интеграция с TG-ботом (уведомления о заказах)
- Базовый SEO
- Срок: 10–14 дней

### Tier 3 — Полная экосистема ($4000–8000)
- Магазин + CRM (Bitrix24 / AmoCRM)
- Email + TG воронка
- Аналитика (GA4 + Яндекс.Метрика)
- Обучение команды
- Срок: 3–4 недели

---

## Почему это нельзя закрыть ChatGPT или Wix самому?

- Технически можно — но большинство не делают из-за страха "сложно" и "дорого"
- Боль не в инструменте, а в **уверенности и времени**
- Ключевая ценность: **сопровождение и снятие тревоги**

## Go-to-market

**Канал привлечения:**
- Telegram (в тематических чатах про бизнес, продажи, малый бизнес)
- Сарафан через первых клиентов
- Кейсы → публикация в TG-каналах про маркетинг/ecommerce

**Оффер для первых клиентов:**
> "Переведу ваш TG-канал на нормальный сайт за 2 недели. Первым 3 клиентам — скидка 30% в обмен на публичный кейс."

---

## Риски

- Низкий средний чек для B2B → нужен поток
- Клиенты часто не знают чего хотят → нужен чёткий бриф
- Много однотипной работы → нужна шаблонизация/автоматизация

---

## Планы реализации

### 🆓 План 0 — Нулевой бюджет ("самый бедный")
**Стек:** Notion + Super.so (бесплатно) ИЛИ GitHub Pages + Ecwid  
**Срок:** 2–3 дня  
**Стоимость:** 0₽/мес (только домен ~200₽/год если нужен)

**Вариант 0A — Notion как сайт:**
1. Создаём красивую Notion-страницу с каталогом (таблица с фото, ценами, описаниями)
2. Публикуем через **Notion Publish** → бесплатный URL вида `notion.site/username`
3. Оплата: **YooMoney** виджет-кнопка — копируем HTML-код и вставляем в Notion через embed
4. TG-бот: @Pochtabot или @payment_bot — клиент пишет в бот, приходит уведомление
5. Без домена — ссылка `notion.site/...`

**Вариант 0B — GitHub Pages (чуть сложнее):**
1. Создаём репозиторий на GitHub
2. Шаблон сайта: **Astro** или **Jekyll** (бесплатные, много тем)
3. Деплой на **GitHub Pages** или **Cloudflare Pages** — бесплатно
4. Каталог товаров: **Ecwid** бесплатный план (до 5 товаров) — вставляем виджет
5. Оплата через Ecwid: ЮKassa / YooMoney
6. TG-уведомления: Ecwid → Zapier (free tier) → TG Bot

**Итого расходы:**
| Статья | Стоимость |
|---|---|
| Хостинг | **0₽** (GitHub Pages / Cloudflare Pages) |
| CMS | **0₽** (Notion / Hugo) |
| Домен | ~200₽/год (необязательно) |
| Оплата | **0%** комиссии платформы + 2-3% ЮKassa |
| Итого | **~0–200₽/год** |

**Ограничения:**
- Notion: нет кастомного домена на бесплатном плане
- Ecwid free: только 5 товаров
- Нет полноценной корзины и checkout без платных планов
- Внешний вид — базовый, не брендированный

**Для кого подходит:**
> Только для старта и проверки гипотезы. Если есть 1-2 продажи в неделю и бюджет ноль. Как только пошли деньги — сразу переходить на Plan A (Tilda ~$20/мес).

---

### 🟢 План A — No-code быстрый старт (для Tier 1–2)
**Стек:** Tilda + ЮKassa + TG Bot API  
**Срок:** 3–5 дней  
**Стоимость инструментов:** ~$20/мес

**Шаги:**
1. Клиент выгружает посты из TG-канала (вручную или через TG API export)
2. Создаём сайт на **Tilda** — блоки, каталог, форма заказа
3. Подключаем **ЮKassa** или **Tinkoff Pay** через встроенный интеграция Tilda
4. Настраиваем **TG-бот** (BotFather + Telepot/python-telegram-bot) → уведомления о заказах в личку владельца
5. Домен + SSL через Cloudflare (бесплатно)
6. Базовый SEO: title, description, sitemap через Tilda

**Плюсы:** быстро, не нужен разработчик, клиент сам обновляет  
**Минусы:** ограниченная кастомизация, Tilda = ~$20/мес навсегда

---

### 🔵 План B — AI-генерация сайта (Lovable / Bolt)
**Стек:** Lovable.dev или Bolt.new + Supabase + Stripe/ЮKassa  
**Срок:** 5–7 дней  
**Стоимость:** ~$30/мес (Lovable Pro)

**Шаги:**
1. Собираем бриф: структура каталога, цвета, референсы
2. В **Lovable.dev** описываем сайт текстом → AI генерирует React-приложение
3. Подключаем **Supabase** как backend (товары, заказы, пользователи) — Lovable делает это нативно
4. Интегрируем **Stripe** (для зарубежных) или кастомную оплату через ЮKassa API
5. Деплой на **Vercel** (бесплатно для hobby)
6. TG-уведомления: Supabase Edge Function → Telegram Bot API

**Плюсы:** красивый результат быстро, полный код в руках клиента (GitHub)  
**Минусы:** требует базового понимания от клиента, платный Lovable для серьёзных проектов

---

### 🟡 План C — Shopify (для реального магазина)
**Стек:** Shopify + TG-бот через Shopify Webhooks  
**Срок:** 7–10 дней  
**Стоимость:** от $39/мес

**Шаги:**
1. Регистрация Shopify + выбор темы (Dawn — бесплатная, минималистичная)
2. Импорт товаров: CSV из Google Sheets → Shopify Import
3. Оплата: **Shopify Payments** (недоступно в РФ) → ЮKassa через приложение или кастомный gateway
4. Настройка **Shopify Webhooks** → TG-бот уведомляет о новых заказах
5. Домен через Cloudflare (CNAME → Shopify)
6. SEO: Shopify встроенный + приложение Smart SEO

**Плюсы:** проверенная платформа, масштабируется, много готовых решений  
**Минусы:** дорого ($39+/мес), ограничения с оплатой в РФ

---

### 🟣 План D — WordPress + WooCommerce (для максимального контроля)
**Стек:** WordPress + WooCommerce + ЮKassa + Elementor  
**Срок:** 10–14 дней  
**Стоимость:** ~$5–10/мес (хостинг) + плагины

**Шаги:**
1. VPS или shared hosting (Timeweb, Beget — от 300₽/мес)
2. WordPress + WooCommerce установка (1 час)
3. Тема: **Astra** или **Kadence** (бесплатные, быстрые)
4. Каталог: вручную или импорт CSV
5. Оплата: плагин **WooCommerce ЮKassa** (официальный, бесплатный)
6. TG-уведомления: плагин **WooCommerce Telegram Bot** или кастомный через WooCommerce Webhooks
7. SEO: **Yoast SEO** (бесплатный)

**Плюсы:** максимальный контроль, дешёвый хостинг, огромная экосистема плагинов  
**Минусы:** нужно обслуживать сервер, обновления, безопасность
